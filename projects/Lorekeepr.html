Overview

Lorekeepr is a powerful tool for writers and storytellers that automatically builds comprehensive story bibles from manuscripts. It combines local and cloud LLM capabilities with advanced NLP processing to extract characters, relationships, timelines, plot threads, and inconsistencies, providing writers with intelligent insights and question-answering capabilities.

Key Features

The application offers a robust extraction pipeline with an intuitive UI featuring manuscript editing, character management, relationship mapping, timeline visualization, thread tracking, inconsistency detection, and an interactive Ask console for grounded questions about the story.

AI Integration: Supports both local LLMs via OpenAI-compatible APIs (tested with Ollama) and cloud providers like OpenAI, with local AI as the default for privacy and cost efficiency.

NLP Processing: Includes optional spaCy-backed character discovery with health checks and self-tests for reliable entity extraction.

Technical Implementation

Built with a TypeScript/React frontend and Node.js/Express backend, Lorekeepr provides cross-platform support for Windows, macOS, and Linux. The backend serves the built frontend as a single publishable server, simplifying deployment.

The system features a full-screen Codex Logs view for switching between Characters, Relationships, Timeline (flowchart), Threads, and Inconsistencies, with seamless integration back to the manuscript editor.

Highlights

  • Automatic story bible generation from manuscript text
  • Dual LLM support (local Ollama + cloud OpenAI)
  • Advanced NLP with spaCy and Stanza fallback
  • Interactive timeline and relationship visualizations
  • Intelligent question-answering grounded in story context
  • Cross-platform desktop application
  • Real-time inconsistency detection
  • Character watchlist and relationship mapping

Development & Deployment

Lorekeepr includes comprehensive setup scripts for Windows (start-windows.bat) and Unix systems (start-unix.sh), with npm scripts for development and production builds. The application supports both development mode with separate frontend/backend servers and production mode with the backend serving the frontend.

Health checks and self-tests ensure NLP components are functioning correctly, with automatic fallback between spaCy and Stanza for maximum compatibility.

Lorekeepr is actively in development, though the majority of features are functional and ready for launch.
